# RAG-система для анализа финансовых документов

RAG (Retrieval-Augmented Generation) система для извлечения информации из PDF-документов и ответов на вопросы на основе найденного контента. Проект разработан в рамках учебного задания по созданию системы вопросно-ответного взаимодействия с документами.

## Описание задачи

Система решает следующие задачи:
1. Извлечение текста из PDF-документов
2. Индексация документов для эффективного поиска
3. Поиск релевантных фрагментов под каждый вопрос
4. Генерация точных ответов на основе найденного контента
5. Формирование выходного файла в требуемом формате

## Особенности

- **Параллельная обработка PDF**: многопоточное извлечение текста из документов
- **Векторный поиск с FAISS**: быстрый поиск релевантных фрагментов
- **Интеграция с Google Gemini**: использование современных LLM для генерации ответов
- **Оптимизация памяти**: пакетная обработка документов при индексации
- **Автоматическая очистка ответов**: извлечение только необходимых значений (числа, True/False, N/A)
- **Сохранение прогресса**: возможность прерывания и возобновления обработки
- **Детектирование квот API**: graceful обработка ошибок превышения лимитов

## 🛠 Технологический стек

- **LangChain** - фреймворк для построения RAG-систем
- **FAISS** - векторная база данных для поиска
- **Sentence Transformers** - эмбеддинги для векторизации текста
- **Google Gemini 1.5 Flash** - генеративная модель для ответов
- **PyMuPDF** - извлечение текста из PDF
- **ThreadPoolExecutor** - параллельная обработка документов

## Использование

1. Установите API ключ Google Gemini (в коде или через переменную окружения):
```python
os.environ["GOOGLE_API_KEY"] = "your-api-key"
```

2. Запустите приложение:
```bash
python main.py
```

3. Результаты будут сохранены в файл `submission_<фамилия>_v<версия>.json`

## Особенности реализации

### Обработка PDF
- Многопоточная загрузка документов
- Вычисление SHA-1 хеша для каждого файла
- Сохранение метаданных (имя файла, номер страницы)

### Индексация
- Пакетная обработка по 1000 документов
- Пропуск пустых документов
- Автоматическое ограничение при большом количестве чанков

### Генерация ответов
- Строгий промпт для получения точных значений
- Пост-обработка ответов (очистка префиксов, нормализация boolean, извлечение чисел)
- Ограничение количества токенов (50) для кратких ответов

### Обработка ошибок
- Graceful обработка ошибок API
- Сохранение прогресса каждые 10 вопросов
- Детектирование превышения квот

## Возможные проблемы и решения

1. **"Missing environment variables"**: Убедитесь, что установлен `GOOGLE_API_KEY`
2. **"No PDF files found"**: Проверьте путь к директории с документами
3. **"Failed to create index"**: Проверьте доступную память, уменьшите количество чанков
4. **API quota exhausted**: Уменьшите количество вопросов или используйте другой API ключ

## Производительность

- Обработка 100 PDF документов: ~2-3 минуты
- Индексация 20000 чанков: ~5-10 минут
- Ответ на вопрос: ~2-3 секунды
- Рекомендуемый объем документов: до 5000 страниц
